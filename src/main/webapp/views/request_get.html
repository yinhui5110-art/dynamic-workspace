요청을 보낼 수 있는 페이지

		Get방식으로 요청 보낸 후 응답데이터 받기
		
		특징 1. GET방식의 경우 URL의 Hearder영역에 요청 시 전달값을 포함해서 요청
		=> 사용자가 입력한 값(데이터들)이 URL에 노출된다
		=> 즉, 로그인이나 회원가입 기능을 구현할 시 GET방식은 부적합하다
		
		특징 2. Header영역은 전송하는 데이터의 길이 제한이 있다
		=>방대한 데이터를 담을 경우 초과된 데이터는 절단된다
		=> 게시글 작성 같은 경우 GET방식이 부적합하다
		
		특징 3. 조회(검색) 기능을 만들 때 적합
		*캐싱한 데이터를 보여준다
		=> 사용자가 입력한 데이터가 URL에 노출되고 응답데이터를 캐싱
		=> 즐려찾기(북마크) == 해당 URL을 등록해놓는 기능
